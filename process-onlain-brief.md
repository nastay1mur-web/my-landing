# Процесс: Онлайн-бриф перед созвоном

> Описание процесса в формате, пригодном для автоматизации.
> Последнее обновление: 2026-03-17

---

## 1. ЧТО ЗАПУСКАЕТ ПРОЦЕСС? (триггер)

**Событие:** Клиент написал в Telegram что-то вроде:
- «Окей, давай созвонимся»
- «Договорились, жду»
- «Когда удобно?»

**Кто запускает:** Ты — вручную, когда замечаешь это сообщение.

**Что нужно знать в момент запуска:**
| Данные | Откуда берёшь |
|---|---|
| Имя клиента | Telegram |
| @username или chat_id | Telegram |
| Тип проекта (квиз / калькулятор / сайт) | Из контекста переписки |
| Дата и время созвона (если уже договорились) | Из переписки |

## 2. ПОШАГОВЫЕ ДЕЙСТВИЯ

### Шаг 1 — Отправка вопросов клиенту
| | |
|---|---|
| **Кто** | Ты |
| **Что** | Пишешь в Telegram: «Чтобы я лучше подготовился, ответь на пару вопросов: 1) Какая ниша? 2) Что должен делать калькулятор? 3) Какой бюджет?» |
| **Время** | 2 минуты |
| **Данные на входе** | Контакт клиента, тип проекта |
| **Данные на выходе** | Отправленное сообщение |
| **Боль** | Каждый раз пишешь одно и то же вручную, забываешь какой-то вопрос |

---

### Шаг 2 — Ожидание ответов
| | |
|---|---|
| **Кто** | Клиент |
| **Что** | Отвечает в свободной форме: может сразу, может через несколько часов. Отвечает частями, вперемешку с другими сообщениями («кстати, ещё вопрос…», «и да, забыл сказать…») |
| **Время** | 5–10 минут у клиента, 2–3 часа ожидания у тебя |
| **Данные на входе** | Вопросы из шага 1 |
| **Данные на выходе** | Несколько разрозненных сообщений в чате |
| **Боль** | Нельзя не смотреть в телефон — вдруг ответил |

---

### Шаг 3 — Напоминание (если нет ответа)
| | |
|---|---|
| **Кто** | Ты |
| **Что** | Проверяешь Telegram каждые 30 минут. Если через 3+ часа ответа нет — пишешь напоминалку вручную |
| **Время** | 2–3 часа в фоне + 2–5 минут на написание напоминания |
| **Данные на входе** | Факт отсутствия ответа |
| **Данные на выходе** | Напоминание клиенту |
| **Боль** | Легко забыть, особенно если несколько клиентов одновременно |

---

### Шаг 4 — Структурирование ответов
| | |
|---|---|
| **Кто** | Ты |
| **Что** | Копируешь ответы из Telegram в Apple Notes или Notion. Убираешь лишнее, приводишь к понятной структуре |
| **Время** | 5 минут |
| **Данные на входе** | Сырые сообщения клиента из Telegram |
| **Данные на выходе** | Структурированный бриф: ниша, продукт, ЦА, бюджет, дедлайн |
| **Боль** | Ручная работа, которую можно делегировать AI |

---

### Шаг 5 — Поиск слепых зон и вопросы на звонок
| | |
|---|---|
| **Кто** | Ты |
| **Что** | Перечитываешь бриф и думаешь: что клиент не упомянул, но это важно? Пишешь список уточняющих вопросов для звонка |
| **Время** | 10 минут |
| **Данные на входе** | Структурированный бриф из шага 4 |
| **Данные на выходе** | Список из 3–7 уточняющих вопросов |
| **Боль** | Легко что-то пропустить; качество анализа зависит от усталости и настроения |

**Типичные слепые зоны, которые всегда надо проверять:**
- Куда идут заявки после квиза? (Telegram, email, CRM?)
- Нужна ли оплата прямо на странице?
- Кто будет обновлять контент после сдачи?
- Есть ли готовый брендинг (цвета, шрифты, логотип)?
- Мобильная версия важна?
- Нужна ли интеграция с CRM или внешними сервисами?

---

### Шаг 6 — Сохранение в файл проекта
| | |
|---|---|
| **Кто** | Ты |
| **Что** | Переносишь данные в папку клиента (Notion / папка на компьютере). Создаёшь файл с брифом и отдельный файл с вопросами на звонок |
| **Время** | 3 минуты |
| **Данные на входе** | Бриф + вопросы из шагов 4–5 |
| **Данные на выходе** | Два файла: `бриф.md` и `вопросы-на-звонок.md` |
| **Боль** | Ещё одна ручная операция, данные иногда теряются |

---

### Шаг 7 — Созвон
| | |
|---|---|
| **Кто** | Ты + клиент |
| **Что** | Проводишь созвон, задаёшь уточняющие вопросы, дополняешь файл проекта по ходу |
| **Время** | 30–40 минут |
| **Данные на входе** | Вопросы из шага 5 |
| **Данные на выходе** | Финальный бриф, готовый к написанию ТЗ |
| **Автоматизировать** | Нельзя — это твоя живая коммуникация |

## 3. ЧТО НА ВЫХОДЕ?

К моменту созвона у тебя должно быть готово:

```
📁 Клиент: Андрей — 20 марта 2026
├── бриф.md              ← структурированные ответы (ниша, продукт, ЦА, бюджет, дедлайн)
├── вопросы-на-звонок.md ← 3–7 уточнений, которые найдёт Claude
└── тз.md                ← заполнится ПОСЛЕ звонка
```

**Бриф содержит:**
- Ниша и целевая аудитория
- Что должен делать продукт (квиз / калькулятор / сайт)
- Куда идут заявки
- Бюджет и дедлайн
- Текущая ситуация (есть сайт или нет, есть брендинг или нет)

**Вопросы на звонок содержат:**
- То, что клиент не упомянул, но важно для оценки проекта
- Сгенерированы автоматически на основе брифа

## 4. ГДЕ МОЖНО СЛОМАТЬСЯ?

| Проблема | Почему возникает | Решение |
|---|---|---|
| Клиент не ответил вообще | Занят, забыл, не увидел | Автонапоминание через 3 ч, потом через 24 ч |
| Клиент ответил в другом чате | Перепутал, написал в общий чат | Всегда давать прямую ссылку на нужный чат |
| Ответы слишком размытые («ну, что-то про бизнес») | Клиент не понял вопрос | Добавить пример к каждому вопросу в шаблоне |
| Ответы пришли частями, растянуты на 5 часов | Клиент занят, отвечает когда вспомнил | Бот копит сообщения по таймеру, не обрабатывает каждое отдельно |
| Два клиента одновременно, данные перепутались | Нет изоляции | У каждого клиента — отдельный thread с уникальным ID |
| Созвон прошёл, а бриф ещё не готов | Клиент ответил поздно | Бот знает дату созвона и присылает бриф за 1 ч до него |
| Claude неверно структурировал ответ | Слишком мало данных | Проверка: если ключевое поле пустое — запросить уточнение у клиента |

## 5. ПРЕДЛОЖЕНИЕ ПО АВТОМАТИЗАЦИИ

### Что автоматизируется, что остаётся ручным

| Шаг | Сейчас | После автоматизации | Экономия |
|---|---|---|---|
| 1. Отправка вопросов | 2 мин руками | 0 мин — 1 команда боту | 2 мин |
| 2. Ожидание ответов | 2–3 ч в фоне | 0 мин — бот слушает сам | 2–3 ч |
| 3. Напоминание | 2–5 мин руками | 0 мин — бот по таймеру | 2–5 мин |
| 4. Структурирование | 5 мин руками | 0 мин — Claude API | 5 мин |
| 5. Слепые зоны | 10 мин руками | 0 мин — Claude API | 10 мин |
| 6. Сохранение | 3 мин руками | 0 мин — бот в Google Sheets | 3 мин |
| 7. Созвон | 30–40 мин | 15–20 мин (лучше подготовлен) | 15–20 мин |
| **Итого** | **~22 мин + 2–3 ч ожидания** | **1 команда + созвон** | **~20 мин + всё ожидание** |

### Архитектура решения

```
Ты: /бриф @andrey_fit Андрей калькулятор 2026-03-20
         │
         ▼
Telegram-бот
  → сохраняет клиента (ID, имя, тип проекта, дата созвона)
  → отправляет Андрею шаблон с вопросами + примерами
         │
         ▼
Клиент отвечает (свободный текст, в любом порядке, частями)
         │
         ▼
Бот копит сообщения в буфер (3–24 ч)
  → по таймеру: если нет ответа через 3 ч — напоминание
  → по команде: /обработать @andrey_fit — запускает обработку досрочно
         │
         ▼
Claude API
  → получает сырые сообщения клиента
  → возвращает JSON: { ниша, продукт, цА, бюджет, дедлайн, текущая_ситуация }
  → находит незаполненные поля → формирует список слепых зон
         │
         ▼
Google Sheets
  → создаёт строку клиента с брифом
  → ссылка сохраняется
         │
         ▼
Бот присылает тебе итог:

✅ Бриф по Андрею готов
──────────────────────
НИША: фитнес-тренер, онлайн
ПРОДУКТ: калькулятор срока похудения
ЦА: женщины 25–40
БЮДЖЕТ: 15 000 ₽ | ДЕДЛАЙН: до конца марта
САЙТ: нет | БРЕНДИНГ: нет

❓ ВОПРОСЫ НА ЗВОНОК:
1. Куда идут заявки после квиза? (Telegram / email?)
2. Нужна оплата прямо на странице?
3. Кто будет обновлять контент?

📊 Google Sheets → [ссылка]
```

### Инструменты

| Задача | Инструмент | Почему |
|---|---|---|
| Твой интерфейс + чат с клиентом | **Telegram Bot API** | Уже в Telegram, ничего нового учить |
| Логика, таймеры, оркестрация | **Python (python-telegram-bot)** или **n8n** | Гибко, бесплатный хостинг на Railway |
| Структуризация и слепые зоны | **Claude API** (claude-sonnet-4-6) | Лучше всего понимает контекст, легко настраивать промпт |
| Хранение брифов | **Google Sheets API** | Бесплатно, открываешь с телефона, легко поделиться |
| Напоминания | **APScheduler** (Python) / **n8n Scheduler** | Простая задача, уже встроено в оба варианта |

### Команды бота (интерфейс)

```
/бриф @username Имя тип_проекта дата_звонка  — запустить процесс для клиента
/обработать @username                         — обработать ответы досрочно
/статус @username                             — посмотреть что уже ответил клиент
/список                                       — все активные брифы
```
